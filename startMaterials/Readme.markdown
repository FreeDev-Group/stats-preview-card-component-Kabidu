#  Project Challenge: Stats Preview Card Component

Welcome to your next frontend challenge! This repository contains the starter files for the **Stats Preview Card Component**. Your goal is to build a responsive, professional card based on the provided design specifications.

##  How to Get the Starter Materials

To begin this challenge, follow these steps to set up your local development environment:

1. Go to the challenge page: [Stats Preview Card Component](https://www.frontendmentor.io/challenges/stats-preview-card-component-8JqbgoU62).
2. Click the **"Start Challenge"** button.
3. Click on **"Download starter"** to get the `.zip` folder.
4. Extract the folder on your computer.

---
##  Project Requirements

To successfully complete this challenge, your solution must meet the following criteria:

* **Responsive Layout:** The component must transition smoothly from a mobile (vertical) layout to a desktop (horizontal) layout.
* **Design Accuracy:** Match the colors, font weights, and spacing provided in the `/design` folder and the `style-guide.md`.
* **The "Purple" Image:** You must apply the purple color overlay to the header image using **CSS only**. Do not edit the image file itself.

---

##  Suggested Workflow

I recommend following these steps to work efficiently:

1.  **Analyze the Design:** Look at the desktop and mobile screenshots. Identify which elements are constant and which change (e.g., text alignment, image position).
2.  **Structure the HTML:** Write your HTML first. Focus on the content hierarchy before adding any styles.
3.  **Define Global Styles:** Set up your colors and typography in your CSS using **CSS Variables** (refer to the `style-guide.md`).
4.  **Mobile-First Styling:** Start by styling for the mobile view. It is much easier to stack elements vertically and then "unstack" them for desktop later.
5.  **Refine & Polish:** Check your margins, padding, and font sizes against the design.

---


##  Getting Started

1.  **Clone** this repository to your local machine.
2.  Open `index.html` in your browser.
3.  Refer to `style-guide.md` for the exact color palette and typography requirements.
4.  **Happy Coding!**
